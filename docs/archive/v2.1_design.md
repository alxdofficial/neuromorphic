# V2.1 design — flat-bank foundation + grammar thesis

Working doc for the V2.1 design discussion (started 2026-05-19). Branched
from `main` at tag `v2-final`. V2 is preserved in `outputs/wave1_v2/` and
in the `src/trajectory_memory_v2/` source tree for reference.

## Why V2.1

V2 measurements (on `composite_v1` val, 800 paired chunks):

| Architecture | Mean NLL | Memory Δ | Status |
|---|---:|---:|---|
| Llama no-context | 4.778 | — | floor |
| Llama full 8-fact context | 3.731 | — | ceiling |
| **flat-bank** (mem on) | **2.628** | **−1.231** | **only working memory architecture** |
| V1.5 trajectory + contrastive | 2.026 | +0.306 | memory HARMS |
| V2 vocab-trajectory | 1.533 | ≈ 0 | memory DOES NOTHING |

V1.5 was killed by the fixed graph topology (reach 0.12 ceiling). V2's
dynamic-edge fix raised reach to 0.27 and gave excellent routing on
state-tracking tasks (boxes rw_target_hop = 0.92), but the readout
structurally bypassed edge_state — only `concept_ids[visited]` reached
the bridge. So routing worked, content didn't flow.

The empirical conclusion: **flat-bank is the only design where memory
contributes meaningfully to NLL**. The thesis (nodes-as-vocabulary,
edges-as-grammar, lifelong learning) is not refuted — V2 just had a wiring
bug — but the simplest working primitive (top-K continuous attention)
should be the starting point we build the thesis features back onto.

## Design goals for V2.1

1. **N ≈ 4096 fixed-vocabulary nodes** — preserves the vocabulary anchor.
2. **Free reach for entry AND inner routing** — no fixed K_max=32
   adjacency. Both reads and writes can in principle touch any cell.
   (V1.5's reach=0.12 ceiling was structural; V2's K_max=32 with dynamic
   eviction was a compromise. V2.1 lifts the constraint entirely.)
3. **Continuous attention as the primary readout mechanism** — what
   flat-bank does. The readout is a weighted sum over many cells, not a
   discrete trajectory of 32 visited points.
4. **Edges as relational structure** — preserve the grammar thesis. Edges
   carry per-pair content; the attention can be enriched by relational
   information between cells.
5. **Hebbian plasticity for lifelong learning** — the actual learning
   signal at the edge level: "cells that fire together wire together,"
   updated via co-activation patterns at inference time.
6. **Simple before clever** — V2 had W-TinyLFU eviction + NPMI plasticity
   + SimVQ vocab + Hopfield-tied projectors + per-hop contrastive +
   step_query MLP. V2.1 should start with the minimum that works and only
   re-add complexity that has measurable payoff.

### Needs adaptation (concept applies, definition changes for continuous attention)

These were defined assuming discrete walker visits. For V2.1's continuous
attention:

| V2 metric | V2.1 analog |
|---|---|
| `rw_target_all` / `rw_target_entry` / `rw_target_hop` (set Jaccard over 32 visited cells) | **Attention-distribution RW alignment** = `<read_attention, target_write_attention>` cosine, treated as distribution similarity. `rw_target_lift` = (sim vs target) − (sim vs distractor mean). Same semantic information, applicable to continuous attention. |
| Reachability per hop | Becomes trivially 1.0 (every cell always reachable in flat attention). Drop. |
| `r_uf` (routing uniformity), `w_uf` | **Effective number of attended cells** = `exp(entropy(attention))`. Sharp attention → small effective count; diffuse → large. |
| Cross-question read Jaccard | **Cross-question attention divergence** = 1 − cosine(attn_q1, attn_q2). |
| Lifetime cell utilization (% ever visited) | **Lifetime attention coverage** — % of cells that ever received > threshold attention. |

### V2-specific (drop entirely if V2.1 doesn't have edge slots)

These are tied to V2's dynamic-edge mechanism:
- `edge_state_norm`, `n_active_edges`, `edge_active_fraction`,
  `mean_edge_specificity`, `mean_edge_age`, `mean_visit_count`
- W-TinyLFU effectiveness ratio, eviction telemetry
- Per-hop reachability decomposition

If V2.1 keeps an edge concept (e.g., as a relational overlay on the
flat-bank attention), we re-introduce edge telemetry then.

### New metrics worth adding for V2.1

| Metric | Why |
|---|---|
| Attention entropy per write/read | Distinguishes "sharp routing to a few cells" (concept-like behavior) from "diffuse soft retrieval" (smoothing) |
| Edge-state Hebbian co-activation rate | Hebbian update telemetry: how often are cell-pairs co-active enough to trigger plasticity? |
| Top-K mass fraction | The cumulative attention weight on top-K cells — quantifies "is attention concentrated or spread thin?" |
| Across-chunk cell stability | For a given task family, does the same input pattern land on the same cells across chunks? (Stability metric for the vocabulary anchor claim.) |

## Reference baselines V2.1 must beat or match

To claim V2.1 is working, on `composite_v1` val (paired with-mem vs no-mem):

| Target | Threshold | Source |
|---|---:|---|
| Memory Δ NLL/tok | ≤ −0.5 | Must beat V1.5's +0.306 and V2's ~0; should approach flat-bank's −1.231 |
| Memory Δ first-token NLL | ≤ −1.0 | Cleanest retrieval signal |
| Memory Δ on biographical (the named-entity test) | ≤ −0.5 | V2 was 0; V1.5 was −0.58; flat-bank was −1.08 |
| Memory Δ on passphrase (verbatim recall) | ≤ −2.0 | V2 was 0; flat-bank was −4.07; anything < −2 is real retrieval |
| Cross-question attention divergence | ≥ 0.2 | Reads should change with question |
| Attention entropy (read) | informative but task-dependent | sharp on retrieval tasks; could be diffuse on aggregation |

If V2.1 beats all of these we'd have an architecture that does what
flat-bank does AND preserves the thesis structures (vocabulary anchors,
relational edges, Hebbian plasticity). That's the win condition.

## Open design questions (to resolve before implementation)

1. **What carries per-event content?** In flat-bank, content lives in
   the attention weights at retrieval time — no persistent state beyond
   trained cell embeddings. To preserve "lifelong learning," V2.1 needs
   *some* persistent per-event state. Options:
   - Per-cell mutable state (V1.5-style) — has known dilution problem
   - Per-edge state (V2-style) — has known named-entity problem
   - Per-cell state with session-local fast weights — the option we
     started discussing (slow concept_ids + fast session-local state).
     Resets between chunks; avoids cross-chunk dilution.

2. **What's the routing decision?** Free-reach options:
   - Top-K continuous attention over all N cells (pure flat-bank)
   - Top-K with edge-weighted scoring (attention biased by cell-pair
     edges)
   - Sparse top-K with discretized writes (writes touch K cells; reads
     soft-attend over all N)

3. **Hebbian plasticity scope.** What does "Hebbian" update? Options:
   - Edge_state for cell pairs that co-activate above threshold
   - Cell embeddings themselves (via slow drift)
   - The attention bias / scoring function

4. **Write mechanism.** Where do writes deposit content?
   - Into per-cell fast state (the session-local node fast weights idea)
   - Into edge_state via Hebbian co-activation
   - Both

5. **Read mechanism.** What does the read attend over?
   - Concept_ids alone (V2's bug — content doesn't reach bridge)
   - Concept_ids + per-cell fast state
   - Concept_ids + edge_state of edges leaving the cell (relational
     enrichment)
   - All three

6. **Reset semantics.** What gets reset between chunks?
   - Fast cell state (definitely)
   - Edge state (probably not — that's the lifelong-learning piece)
   - Anything else?

7. **Bridge MLP.** Keep V2's mem_inject design (it works as far as we
   tested) or redesign?
   - Keep — `MemInjectLayer` from `src/pretrained/` is reused across V1
     and V2 and works
   - Open question is just what `memory_fn` returns (the readout
     mechanism above), not the bridge itself

## Implementation order (when design is locked)

1. **Sanity-check edge_state update algorithm** on V2 (the current
   implementation has primacy bias from global visit_count, unused
   specificity, bf16/fp32 mixing). Fix or replace.
2. **Build V2.1 minimum**: flat-bank top-K attention + concept_ids
   (vocab) + per-cell fast state (reset per chunk) + Hebbian-updated
   edge_state.
3. **Port the diagnostic suite** with the V2.1-adapted metrics.
4. **Smoke test** at 1-2K steps. Expected: memory contribution
   becomes negative (memory helps), even before tuning.
5. **Full retrain** 10K steps on composite_v1. Measure against the
   target thresholds above.
6. **Iterate** if any task family is broken.
