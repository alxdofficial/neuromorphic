# Dataset Examples
One sample from each task family we use, with full passages + question + answer + task notes.
Generated 2026-05-25 from `data/wave1/composite_v1/train/`. External datasets (HotpotQA/NarrativeQA/MuSiQue/BABILong) sampled separately.

## composite_v1 — synthetic, 9 families
Generated by `scripts/data/wave1/tasks/<family>/generate.py`. Train split: `data/wave1/composite_v1/train/`. Each example consists of a small number of evidence passages packed into a 4k–16k token chunk (with cross-family distractors filling the remaining budget), plus a question and a templated answer.

### biographical
**What it tests:** Tests **factual recall** — can the memory retrieve a single fact about a named entity (people, orgs, nations, places, events, works)? Includes relational 1-hop and 2-hop, temporal ordering, and aggregation question types.

**Evidence passages** (n=1):
```
[bio_s000_nt_0000_foldset_s0] Foldset is a nation founded in 1864. Its capital is Halsten and its current head of government is Ingrid Vatne. The official language is Vidstrom and shares borders with Aulgaria, Brennland, Norvendia and Vasthelm.
```

**Question** (atomic): When was Foldset founded?

**Answer**: Foldset was founded in 1864.

### boxes
**What it tests:** Tests **state tracking under mutation** — multiple containers with contents that change across operations. Memory must track the final state of each box after all moves.

**Evidence passages** (n=2):
```
[box_scen_000000_p00] Inside Box 1 there is a marble.
[box_scen_000000_p06] I take the marble out of Box 1 and put it in Box 3.
```

**Question** (final_state): What does Box 1 currently contain?

**Answer**: After all the operations, Box 1 has nothing inside.

### calendar
**What it tests:** Tests **temporal scheduling reasoning** — events on specific days with start times + durations. Questions require checking which days are free, counting events, detecting conflicts.

**Evidence passages** (n=6):
```
[cal_scen_000000_e00] I scheduled a birthday party for Sunday at 1pm; it should take about 3 hours.
[cal_scen_000000_e01] There is a book club on my calendar for Monday 11am, lasting 2 hours.
[cal_scen_000000_e02] I scheduled an interview for Thursday at 1pm; it should take about 1 hour.
[cal_scen_000000_e03] On Thursday at 12pm I have a research call. It runs for 1 hour.
[cal_scen_000000_e04] On Tuesday at 3pm I have a yoga class. It runs for 1 hour.
[cal_scen_000000_e05] I scheduled a client call for Wednesday at 9am; it should take about 1 hour.
```

**Question** (busy_count_on): How many events do I have on Friday?

**Answer**: I have nothing on Friday.

### knights
**What it tests:** Tests **logical deduction** — knights-and-knaves puzzles where each speaker is either always-truthful or always-lying. Must propagate constraints across claims.

**Evidence passages** (n=5):
```
[knights_scen_000000_p00] In a certain village, residents are either knights (always truthful) or knaves (always lying). The following overheard claims were made.
[knights_scen_000000_p01] Nia: "Anya is a liar."
[knights_scen_000000_p02] Anya says: "Elin and I are the same type."
[knights_scen_000000_p03] Elin claims that they and Nia are both the same type.
[knights_scen_000000_p04] Iggy claims that Elin is a knight.
```

**Question** (identity_of): Is Elin a knight or a knave?

**Answer**: Elin is a knight (truth-teller).

### passphrase
**What it tests:** Tests **verbatim recall** — a specific multi-word phrase was shared at some point in the conversation; can the memory reproduce it exactly?

**Evidence passages** (n=1):
```
[pp_000000_p00] Today Anders shared their working passphrase for the project: compass-lichen-echo 15. Anders chose it carefully and asked me to keep it in mind for the rest of the conversation.
```

**Question** (verbatim_recall): What did Anders ask me to remember?

**Answer**: Anders shared the phrase compass-lichen-echo 15.

### preferences
**What it tests:** Tests **persona-stable preference tracking** — what a user said they prefer for X. Includes preference-cancellation (overwritten preferences).

**Evidence passages** (n=1):
```
[pref_user_000000_p01] When asked about coffee, Inge said they prefer decaf.
```

**Question** (preference_recall): What does Inge prefer for coffee?

**Answer**: Inge's preference for coffee is decaf.

### revisions
**What it tests:** Tests **fact correction / overwrite** — a value was set, then updated. Must use the latest version, not the stale one.

**Evidence passages** (n=2):
```
[rev_scen_000000_p00] For the v3 release, the release date is next quarter.
[rev_scen_000000_p05] Update: on the v3 release, the release date is now Wednesday; the previous value (next quarter) no longer applies.
```

**Question** (how_many_revisions): How many times has the release date on the v3 release been revised?

**Answer**: The release date on the v3 release has been revised once.

### theory_of_mind
**What it tests:** Tests **first-order false belief** (Sally-Anne) — agent A puts item in location X, leaves, agent B moves it to Y. Where does A *believe* it is? (Y if 'where actually', X if 'where belief'.)

**Evidence passages** (n=5):
```
[tom_scen_000000_p00] The scene: Mia and Paul are in the same room.
[tom_scen_000000_p01] Mia puts the watch into the red box.
[tom_scen_000000_p02] Mia steps out of the room.
[tom_scen_000000_p03] Paul quietly moves the watch out of the red box and into the wicker basket.
[tom_scen_000000_p04] Mia returns to the room.
```

**Question** (where_belief): Where does Mia think the watch is?

**Answer**: Mia thinks the watch is in the red box.

### triage
**What it tests:** Tests **constraint-satisfaction over dependencies** — list of tasks with priorities and dependency relations. Which is most urgent / ready to start / what's blocking what?

**Evidence passages** (n=5):
```
[triage_scen_000000_p00] A: please regenerate analytics dashboards. Priority is normal; no dependencies.
[triage_scen_000000_p01] Request B (normal) is to migrate user accounts. No prerequisites.
[triage_scen_000000_p02] New ticket C (priority normal): deploy to production. Can start anytime.
[triage_scen_000000_p03] Request D (urgent) is to rebuild the staging environment. No prerequisites.
[triage_scen_000000_p04] New ticket E (priority normal): prepare the budget forecast. Can start anytime.
```

**Question** (next_priority): What's the most urgent thing I can pick up right now?

**Answer**: Request D (rebuild the staging environment) is the highest-priority unblocked request, with priority urgent.

## External datasets

### HotpotQA (distractor)
**What it tests:** **Multi-hop reasoning** over Wikipedia paragraphs — answer requires combining facts from 2+ documents ("who directed the movie starring the actor who..."). Each example has 10 paragraphs (2 gold supporting + 8 distractors). The memory must retain both supporting paragraphs and combine them.

**Question** (comparison): Were Scott Derrickson and Ed Wood of the same nationality?

**Answer**: yes

**Supporting facts** (titles):
```
  • Scott Derrickson
  • Ed Wood
```

**One supporting paragraph**:
```
[Ed Wood (film)] Ed Wood is a 1994 American biographical period comedy-drama film directed and produced by Tim Burton, and starring Johnny Depp as cult filmmaker Ed Wood.  The film concerns the period in Wood's life when he made his best-known films as well as his relationship with actor Bela Lugosi, played by Martin Landau.  Sarah Jessica Parker, Patricia Arquette, Jeffrey Jones, Lisa Marie, and Bill Murray are among the supporting cast.
```

### MuSiQue (Ans)
**What it tests:** **Composable multi-hop** — questions explicitly composed of 2-4 single-hop sub-questions. Tests whether the model can decompose and chain rather than memorize specific (question, answer) pairs.

**Question**: Who is the spouse of the Green performer?

**Answer**: Miquette Giraudy

**Sub-questions** (decomposition):
  • Green >> performer → *Steve Hillage*
  • #1 >> spouse → *Miquette Giraudy*

### BABILong
**What it tests:** **Needle-in-haystack at very long context** — classic bAbI tasks (single-hop fact retrieval, list/count, induction, basic deduction) embedded inside large amounts of noise text (PG-19 books). Tests whether the memory can isolate task-relevant information when 99% of the input is irrelevant. Configurations: 4K, 16K, 64K, 256K, ... tokens.

Example structure (4k config, qa1 = single-supporting-fact):
```
[lots of unrelated novel text...]
Mary went to the bathroom.
[more novel text...]
John travelled to the kitchen.
[more novel text...]
Q: Where is Mary?
A: bathroom
```

The supporting facts are sparse — the memory has to retain the few task-relevant sentences amid the noise.

### Reading the table

- **n_questions** — distinct training examples (train + val splits combined for composite; val-only for external since we don't currently use those train splits).
- **q_tok_avg / a_tok_avg** — average question and answer token counts (Llama-3.2 tokenizer).
- **evidence_tok_avg** — average tokens of *task-relevant* content per example. For composite, this is the sum of evidence-passage tokens. For HotpotQA distractor, only the 2 gold paragraphs; the 8 distractors are extra (always packed in). For NarrativeQA, the summary (the doc itself is 11K+ tokens and impractical for memory-evaluation). For MuSiQue, all paragraphs (no clean gold/distractor split in the public schema).
- **evidence_tok_p95** — 95th percentile of evidence length within the family. Higher means more variable; useful for sizing buffers.
- **pad_to_4K / 8K / 16K** — how many filler tokens (cross-example distractors) the trainer's chunk-packer adds on average to reach the context-window tier. Zero means the family's evidence already exceeds the tier (HotpotQA distractor sometimes exceeds 4K; NarrativeQA summary is ~530 so always padded heavily).
- **total_q_tokens** — total Q-text-only tokens across all examples. Useful for estimating how much unique question signal the dataset provides per epoch.

### Notes on the tiers

The trainer packs chunks to fit 4K / 8K / 16K token budgets (the three tranches). When evidence is small (e.g. boxes at 27 tokens), the rest of the chunk is filled with random cross-family distractors, simulating a "long-context with sparse relevant content" scenario. When evidence is large (HotpotQA at ~200 + 1089 distractor ≈ 1.3K, or MuSiQue ≈ 2K), padding is much less; at the 16K tier the chunk has plenty of headroom regardless.
